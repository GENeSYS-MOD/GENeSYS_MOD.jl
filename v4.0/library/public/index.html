Types

GENeSYSMOD.CSVResult — Type

CSVResult - Raw results will be printed in CSV format. Each variable will have its CSV file named after the variable.

source
GENeSYSMOD.NoInfeasibilityTechs — Type

NoInfeasibilityTechs - type to use when the infeasibility techs are not needed.

source
GENeSYSMOD.NoRawResult — Type

NoRawResult - The raw results will not be printed.

source
GENeSYSMOD.OneNodeSimple — Type

OneNodeSimple - The dispatch is conducted for the considered_region in isolation from bordering regions.

source
GENeSYSMOD.OneNodeStorage — Type

OneNodeStorage - The dispatch is conducted for the considered_region while the interaction with the rest of the system is modeled as storages.

source
GENeSYSMOD.TXTResult — Type

TXTResult - Raw results will be printed in TXT format. A single txt file containing all values will be printed. This type as an argument filename used to define the name to use for the txt file. By default the argument takes the value "results".

source
GENeSYSMOD.TXTandCSV — Type

TXTandCSV - Raw results will be printed in both TXT and CSV formats.

source
GENeSYSMOD.TwoNodes — Type

TwoNodes - The dispatch is conducted for the considered_region and another dummy region consisting of an aggregation of the other regions.

source
GENeSYSMOD.WithInfeasibilityTechs — Type

WithInfeasibilityTechs - type to use when infeasibility techs are desired in the model. Typically they are used to help troubleshoot infeasibility as they provide a relaxation that allow the model to solve and to find the origin of theinfeasibility in the results.

source
